# MiCS-5524 — CO, Alcohol & VOC Gas Sensor

**Status: Integrated — wired to A0, data flowing to server.**

- Product: [Adafruit MiCS5524 Breakout](https://www.adafruit.com/product/3199) (~$5)
- Guide: [Adafruit Learn Guide](https://learn.adafruit.com/adafruit-mics5524-gas-sensor-breakout)

## How It Works

Heated metal-oxide semiconductor (MOS) sensor. A resistive element (Rs) changes resistance in the presence of reducing gases — Rs **decreases** when gas is detected. The Adafruit breakout forms a voltage divider between Rs (100k–1500k in clean air) and a 10k load resistor (RL), producing an analog voltage:

```
Vout = 5V × RL / (Rs + RL)
```

Clean air: Rs is high (~500k), Vout is low (~0.1V, ADC ~120).
Gas detected: Rs drops (e.g. 50k), Vout rises (~0.8V, ADC ~1000).

Higher ADC reading = more gas detected.

## Wiring

```
MiCS5524    →  Nano ESP32
5V          →  VBUS (5V)
GND         →  GND
AOUT        →  A0 (GPIO1, ADC1 — WiFi-safe)
EN          →  GND (always enabled)
```

A0 is on ADC1 (GPIO 1–10), which works when WiFi is active. ADC2 (GPIO 11–20, pins A4–A7) gives garbage readings with WiFi on.

## Breakout Board Components

| Component | Value | Purpose |
|-----------|-------|---------|
| R1 | 82Ω 1% | Drops 5V to ~2.4V heater voltage |
| R2 | 10kΩ | Load resistor (voltage divider with Rs) |
| R3 | 10kΩ | Pull-up for MOSFET gate (default off) |
| Q1 | DMG3415U (P-ch) | Power switch controlled by EN pin |

No level shifting on AOUT — output depends on Rs/RL ratio. In normal air the output stays well under 3.3V. Under extreme gas exposure Rs could drop low enough to push AOUT above 3.3V, but this is unlikely in indoor monitoring. A 3.3V zener clamp on AOUT would add safety if desired.

## Reading

```cpp
int raw = analogRead(A0);              // 0–4095 (12-bit ESP32)
float voltage = (raw / 4095.0) * 3.3;  // 0–3.3V

// Compute sensor resistance from voltage divider
float rs = 10000.0 * (5.0 - voltage) / voltage;  // RL = 10k, Vs = 5V
float rs_r0 = rs / r0_baseline;                   // ratio vs clean-air baseline
```

## Detectable Gases

| Gas | Range (ppm) | Notes |
|-----|-------------|-------|
| Carbon Monoxide (CO) | 1–1000 | Primary target for indoor air quality |
| Ethanol (C2H5OH) | 10–500 | Cleaning products, hand sanitizer |
| Hydrogen (H2) | 1–1000 | |
| Ammonia (NH3) | 1–500 | Cleaning products |
| Methane (CH4) | >1000 | Natural gas leaks |
| Propane / Iso-Butane | >1000 | |

The sensor **cannot distinguish** which gas is present — it responds to all reducing gases. Cross-sensitivity is inherent to MOS sensors.

## Warmup

| Phase | Duration | Notes |
|-------|----------|-------|
| Minimum | ~1 second | Heater on, readings start |
| Baseline stabilization | **3 minutes** | Wait before trusting readings |
| First-use burn-in | **24 hours** | Continuous operation before calibration |

The firmware skips readings for the first 3 minutes after boot.

## Calibration

True PPM conversion is impractical without calibration gas. The datasheet provides log-log sensitivity curves (Rs/R0 vs ppm per gas at 25°C/50% RH), but:
- Response is non-linear
- Cross-sensitive to multiple gases
- Temperature and humidity dependent
- Requires known gas concentrations for R0 baseline

**For Koffing, we use raw ADC values as a relative indicator.** Track trends over time — a sudden rise means air quality is degrading, regardless of which specific gas caused it. The Grafana dashboard shows threshold lines at 300 (yellow) and 800 (red) ADC counts as starting points to tune.

To establish a baseline: note the typical ADC reading in clean air after 24h burn-in. Significant deviation from that baseline indicates gas presence.

## Electrical Specs

- Input voltage: 4.9–5.1V DC
- Heater voltage: 2.4V (via 82Ω series resistor)
- Heater current: 25–35 mA (32 mA typical)
- Heater power: 71 mW
- Sensor resistance in air: 100kΩ–1500kΩ
